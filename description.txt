8. Git & GitHub Explanation
What is Git?
Git is a version control system - it tracks changes to your code like save checkpoints.

Without Git:

my_project_v1.js
my_project_v2.js
my_project_FINAL.js
my_project_FINAL_ACTUAL.js    // ğŸ˜… Confusing!
my_project_FINAL_ACTUAL_2.js
With Git:
One file: my_project.js
But Git remembers every version and change!
Key Git Concepts
1. Commit (Save checkpoint)
git add .                      # "I'm done with these changes"
git commit -m "Added login"    # "Save with message 'Added login'"
2. Branch (Separate work area)
git checkout -b feature/borrow   # Create new branch to work on borrow feature
# Make changes...
git commit -m "Added borrow feature"
git checkout main                # Switch back to main
git merge feature/borrow         # Combine borrow feature into main

3. Push (Upload to GitHub)
git push origin main           # Upload commits to GitHub
Common Git Workflow for Your Project

# Create Pull Request on GitHub â†’ Get review â†’ Merge to main

# Now main branch has all your code
git checkout main
git pull origin main
